# OCSplus UI 设计

## 当前状态

本文档为 UI 占位文档，后续由 UI Agent 根据 `PRD.md` 和现有 Plus 页面补全。

## 初步 UI 目标

OCSplus UI 应该比原 OCS 更适合普通用户使用：

1. 首页清晰展示当前状态。
2. 主要入口一眼可见。
3. 高级功能可以隐藏到原版入口。
4. 配置页简单直接。
5. 错误和运行状态要可见。

## MVP 页面结构

### 1. 今日学习 / 首页

包含：

- 学习概览
- 网课浏览器入口
- OCS 控制台
- 任务提醒
- 原版入口

### 2. 网课浏览器

包含：

- 浏览器路径配置
- 新建并启动浏览器
- 最近浏览器列表
- 浏览器管理入口
- 环境修复入口

### 3. 设置中心

包含：

- 浏览器路径
- 基础偏好
- 模型配置入口
- 高级设置入口

### 4. 原版高级入口

包含：

- 原浏览器管理
- 原脚本管理
- 原资源管理
- 原设置入口

## UI 约束

1. 不为未实现功能制造“已经可用”的错觉。
2. 占位模块必须写清楚“开发中”。
3. MVP 阶段不要堆太多复杂页面。
4. 优先保证桌面端可用，不优先适配移动端。
