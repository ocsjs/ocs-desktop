# OCSplus 产品需求文档

## 1. 项目定位

OCSplus 是基于 `ocs-desktop` 的桌面端学习辅助整合项目。

目标是在保留原 OCS 桌面端能力的基础上，增加一个更清晰的 Plus 工作台，用于统一管理：

- 网课浏览器
- 原 OCS 脚本能力
- 学习进度概览
- 任务提醒
- AI 题目助手
- 本地题库/错题能力
- 模型与配置管理
- 运行日志与状态反馈

## 2. 第一版 MVP 目标

第一版只追求可运行、可配置、可验证，不追求完整自动化。

MVP 必须做到：

1. 软件可以正常启动。
2. 可以进入 OCSplus 首页。
3. 可以看到 Plus 模块导航。
4. 可以配置浏览器路径。
5. 可以保存基础配置。
6. 可以显示学习/任务/脚本的基础状态。
7. 可以保留并进入原版 OCS 功能入口。
8. 可以显示运行日志或状态提示。

## 3. 暂不做的内容

第一版暂不做：

1. 全自动刷课闭环。
2. 多平台完整适配。
3. 完整 AI 答题自动化。
4. 云同步。
5. 账号体系。
6. 商业化功能。
7. 大规模重构原 OCS 架构。

## 4. 核心用户场景

### 场景 1：启动软件

用户打开 OCSplus，能进入一个清晰的桌面端工作台，而不是直接面对复杂原版页面。

### 场景 2：配置浏览器

用户可以填写或选择本地浏览器路径，并保存配置。

### 场景 3：进入网课浏览器

用户可以从 Plus 首页进入网课浏览器，并继续复用原 OCS 的浏览器管理和脚本能力。

### 场景 4：查看学习状态

用户可以在首页看到学习进度、任务提醒、脚本状态等概览信息。

### 场景 5：回到原版功能

当 Plus 功能不完整时，用户仍可以进入原版 OCS 高级入口完成操作。

## 5. 成功标准

MVP 成功标准：

1. `pnpm dev` 可以启动开发环境。
2. Plus 首页可访问。
3. 原 OCS 核心入口没有被破坏。
4. 基础配置可以保存并再次读取。
5. 用户能理解每个主要入口的用途。
6. 后续 Developer Agent 可以按 `docs/Tasks.md` 逐项开发。
