# OCSplus 任务清单

## 任务状态说明

- `P0`：当前最重要，必须优先完成
- `P1`：MVP 必需，但可排在 P0 后
- `P2`：增强功能，MVP 后再做

状态：

- `todo`：未开始
- `doing`：进行中
- `done`：已完成
- `blocked`：阻塞

### T017：多会话内嵌浏览器与右侧脚本控制台适配（用户新增优先需求）

- 优先级：P0

- 用户决策：内嵌浏览器支持多个受控课程会话；右侧控制台同步脚本运行状态；将可安全替代的原 OCS 浮层消息迁移到控制台；允许受控脚本注入。
- 非目标：不隐藏登录、授权、权限、安全错误、人工确认等关键浮层；不放开任意第三方脚本/网址；不接入自动答题、自动提交。

  3. 右侧控制台显示真实会话/脚本/浮层迁移事件，不伪造“正在刷课”。
  4. 仅明确允许的非关键原 OCS 提示可被右侧控制台替代；登录、权限、错误和人工确认继续可见。

  - [x] D017-4a：受控注入候选、主进程二次校验、一次性 lease 与审计。（App 19/19、Web 14/14、最终安全审查 PASS）

### T014：右侧控制台原生适配脚本运行状态

- 优先级：P1
- 状态：done
- 建议 Agent：Research / Architect / Developer Agent
- 目标：右侧“网课控制台”不再只显示静态开关，而是原生适配 OCSplus 的脚本运行状态与关键流程状态。
- 用户要求：可以参考原版 OCS 的交互和状态含义，但不要照搬原版代码；需要做 OCSplus 原生适配。
- MVP 范围：优先做状态显示和事件流，不直接做完整自动刷课闭环。
- 需要展示的状态示例：
  1. 脚本待命中 / 注入中 / 已注入 / 运行中 / 暂停 / 异常。
  2. 当前课程 / 当前章节 / 当前视频或任务。
  3. 开始刷课。
  4. 视频播放中 / 已完成。
  5. 切换下一章节。
  6. 题目检测中 / 需要人工处理 / 暂未接入 AI。
  7. 错误状态与用户可执行操作。
- 验收标准：
  1. 右侧控制台能显示真实脚本运行状态，而不是纯静态假状态。
  2. “开始刷课”“切换下一章节”等流程状态有明确展示。
  3. 与原版 OCS 的能力边界清晰，只复用概念，不复制实现代码。
  4. 不引入完整自动答题、完整自动刷课闭环。
  5. 状态来源、事件接口、失败提示写入 `docs/Architecture.md` 或 `docs/Research.md`。
- 子任务进度：
  - [x] R014：研究原版 OCS 状态含义与 OCSplus 可用状态来源。
  - [x] A014：设计 `OcsRuntimeStatus` 状态模型、事件流和组件边界。
  - [x] D014-1：新增 runtime 状态类型与默认状态。
  - [x] D014-2：新增 runtime 事件定义与轻量事件总线。
  - [x] D014-3：让 `EmbeddedCourseBrowser` 发出真实状态事件。
  - [x] D014-4：改造 `OcsControlPanel` 为状态展示组件。
  - [x] D014-5：补充失败提示与状态来源展示。
